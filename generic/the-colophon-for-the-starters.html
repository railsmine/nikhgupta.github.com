The Colophon for the Starters06 July 2011

So, here I am.. back and revived.. with all you people starin’ at the new me..
yeah..!! master has given me some new skin.. some new soul and some new toys..
and, I know you are, definitely, wonderin’ at how stylish I can be with so much elegance at my disposal, but I am only being modest..
you don’t even know, the major half of the story.. the shear power..
go on.. read on.. how I was born? about my advent, my dogfood and me..
and most specially, if you are interested in developin’ my soulmate ;)

Hey everyone..
I am Nikhil Gupta, a web developer, and a tech explorer, by which I am only trying to emphasis on the fact that anything remotely linked with technology has all my interest in it - be it linux administration, new gadgets, web development or simply, blogging - infact, you can simply throw any tech-junkie (materialistic, or non-materialistic), and I will savour it, like I was waiting for it to come to me..

The Prologue

I started blogging around 2005 with WordPress.com pages, and soon, moved to self-hosted WordPress for the control it provided me on my blog. I enjoyed everything WordPress provided me, but then it was too much of a pain to login to a site, write inside some WYSIWYG editor, losing important content due to some server migration or hardware-failure or anything-else-equally-evil-not-in-my-mind-right-now. And, how can I deny the fact that WordPress was, just, too easy. Where am I to show that I am a developer, who loves coding, where is the damn innovation?

Unfortunately, the blogging scenario has always been in a hiatus for me, for lack of my interest in ‘simply’ blogging. I, even, developed a WordPress CLI Poster for my personal use (it is unfinished, but does the work on Ubuntu based distros). After a really long period of time, when I was, again, searching for something more developer-friendly, I came across this article: blogging like a hacker by Tom Preston Werner.

That was it.. That was, exactly, what I was searching for. And, it was there for so long, yet I never paid attention. It was based on Ruby, which is why I, probably, never got to it, before. But, as they say, You are never late for anything!

The Advent

So, I started understanding what Jekyll really is - how it can help, what plugins were available, how much I could extend the core, and things similar. And, Jekyll, simply, is amazing. Jekyll is a simple, blog aware, static site generator. It takes a template directory (representing the raw form of a website), runs it through Textile and Liquid converters, and spits out a complete, static website. Some of the features I, now (I have made the WordPress => Jekyll shift), enjoy are:

  • A static HTML website - which means responsiveness, and much better load times.
  • An editor of my choice (gedit on Ubuntu platform, notepad++ on Windows) for writing my content - analogous to writing my code.
  • Git revision control for my website and its content, powered by Github - oh! and, please do not ask me what that means - fare enough to say: it is a must for, literally, anything I do, if applicable even remotely.
  • No complicated blogging engines to maintain, update, or spend more than half of my time customizing it to my taste (everytime)
  • Stylize contents the way, I prefer without the hassle of maintaining something which has no connection with what I am writing (nodes? custom post-types?). I do not write awesome posts. I only write my thoughts in a simple narrative way. I only post some code snippets, some photographs, and probably thats it. I do not need a full-blown CMS.
  • Availability of rake tasks, command line, crons, etc. for simple maintenace, configurations, and hence, following the DRY principle while blogging :)
  • Easy deployment, new posts, upgrades, etc. with a single line on my console.
  • Use of MarkDown for writing posts where the source looks and feels like normal content :) which relieves me from the pain of clicking buttons to make my text bold, italic or a hyperlink.
  • Syntax highlighting inbuilt.

Photography?

Alright, so as I mentioned, I only write simple posts, code snippets and post some photographs. Ouch! Photographs? Yeah, that would make my git repository, just, too heavy! We have automatic syntaxt highlighting (with pygments) and hence, that is taken care of.

But, photographs?

hmm.. I do not want to get too heavy. I am really enjoying the agility, I have, master!

Flickr?

Yeah. There is this really nice plugin, that can import Flickr photosets for me!

Yeah, nice.. But, I need something more flexible. I might, at times, want to show off some photosets - other times, simply some photos or maybe some custom search might be required one day, as well.

hmm.. You can code it for me?

Oh! yes. definitely. But, it is ruby.. :(

at least, you can try!!

Oh! hell, yes.. lets, do it..

That is how things went :) And, I am quite satisfied with the end-result. I, now, have an Advanced Flickr Jekyll plugin that does almost all the functionality, I could have wanted (or anyone else). It allows me to post photos, photosets, galleries, custom searches, with optional exif data and most importantly, cached content without making my website repository, any larger in size. The plugin is available at Github

Whats more?

hmm.. I have various rake tasks defined, which help me developing on my Jekyll site, locally - help me publish this website with a single command, re-build the entire site, commit to Github, create new posts, push some tags and what not.
I, even, went overboard and re-arranged the default folder structure of Jekyll to a custom one, which is then again converted to a Jekyll-aware folder structure via rake tasks. This allows me to define various themes for my sites (a theme, thus defined, helps me configure which skin would be shown or use a particular language for my layouts, and even which plugins will be used, etc.) - which in turn, helps me to customize my Jekyll site to even greater extent, e.g. defining site-wide and theme-based plugins. Moreover, the folder structure looks softer on my eyes. For more details, have a look at this Github repository.

And, the epilogue..

To be honest, I have no idea what to write, in the epilogue. But, as is my custom, I do add a ‘prologue’ and an ‘epilogue’ for each of my posts. I would be working on the Jekyll site, here at www.nikhgupta.com for a major part of this month, and some cool goodies are in development. I will write more on Jekyll and its awesome superiority, but enough for now!
Adieus!
